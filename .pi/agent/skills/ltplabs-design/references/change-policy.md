# Change Policy

- Do not create a new component unless the user explicitly authorizes it with `a teresa deixa` or a clearly equivalent approval.
- If a request needs a missing component, stop and produce a UX-UI note with the need, rationale, affected pages, and the closest existing component.
- Prefer adapting an existing component, page, or template when the change can be made without introducing a new API surface.
- Preserve the template structure; do not move or rename folders unless the task explicitly requires it.
- If the docs and implementation disagree, call out the mismatch and ask which source to follow.
- Ask clarifying questions instead of guessing when the docs do not fully specify the behavior.
